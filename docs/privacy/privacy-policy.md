# Privacy Policy (DRAFT — UNAPPROVED)

**Updated:** 2026-07-20  
**Not legal advice. Not in force until counsel-approved.**

## Design intent

Minimize operator custody of profiles, photos, messages, precise location, and preferences. Prefer device-local storage and E2EE.

## Categories (high level)

| Category | Typical custody | Notes |
|---|---|---|
| Account / device identifiers | Control plane as needed | Minimize retention |
| Age eligibility signals | As designed | Fail closed |
| Public profile fields | User-chosen distribution | Not private chat |
| Private messages / media | Device / E2EE peers | Operator cannot read ordinary content |
| Precise location | Not collected for matching | Approximate only if ever offered |
| Reports / safety evidence | Sealed vault | Need-to-know; audit access |
| Crash / diagnostics | Optional, minimize | No message content by default |

## User rights (placeholders)

Access, correction, deletion, export, and complaint channels: `CHANGE_ME` before public beta.

## Honest limits

Screenshots, peer copies, and law-enforcement process may still expose content. Decentralization does not erase all legal duties.

See `docs/privacy/data-map.md`, `docs/privacy/dpia-outline.md`.
